4. Installation

4.1 Environnement nécessaire

Pour réussir la compilation de Sylpheed, vous avez au moins besoin :

  • D'un système d'exploitation UNIX compatible POSIX ou un système d'exploitation similaire i.e. Linux, FreeBSD, Solaris...
  • GTK+ 1.2.6 ou ultérieur
  • d'un compilateur ANSI C récent ( gcc 2.7.2.3 devrait aussi fonctionner). Note : le compilateur C de Sun ne compilera pas Sylpheed.
Optionnellement, vous avez besoin :
  • de la bibliothèque Imlib ou gdk-pixbuf (pour la visualisation intégrée des images)
  • de la bibliothèque libcompface (pour X-Face)
  • de la bibliothèque libjconv (pour I18N)
  • de la bibliothèque de cryptographie GPGME (pour GNUPG)
  • de la bibliothèque OpenSSL (pour le support de SSL)
  • de la bibliothèque OpenLDAP (pour une connexion avec un serveur LDAP)
  • IPV6 peut être utilisé (si votre système le prends en compte)
  • du logiciel Jpilot (pour utiliser un carnet d'adresses Palm)
  • de l'équivalent de GTK+-devel et Xfree86-devel. (Sinon ./configure échouera.)
Note : Sylpheed peut être instable suivant l'environnement. Dans la mesure du possible, veuillez utiliser les versions les plus récentes des bibliothèques.

4.3 Éléments et points clés de l'installation

  • Téléchargez la dernière version de Sylpheed à partir de http://sylpheed.good-day.net/
  • Décompressez les fichiers sources dans un nouveau répertoire :
  • tar -xvfz sylpheed-.tar.gz(pour tar.gz)
  • tar -xvfI sylpheed-.tar.bz2 (pour tar.bz2)
  • cd vers le nouveau répertoire sylpheed
  • Lancez ./configure pour configurer votre environnement et créer les Makefiles d'installation
  • Lancez le programme make qui compilera les sources. Ne passez pas à cette étape si vous rencontrez des erreurs fatales dans l'étape précédente. Les erreurs fatales indiquent que des éléments manquent de manière à pouvoir compiler Sylpheed correctement.

Si le processus make est terminé, lancez make install (en tant que root), pour installer le programme et les fichiers de support.

4.4 Installation sur Linux en utilisant RPM

En tant que root, lancez la commande rpm -ivh sur le fichier RPM que vous avez téléchargé. Dès que ce processus est fini, vous pouvez lancer Sylpheed comme utilisateur.

4.5 Installation sur Debian Linux

En ayant une connexion internet :

apt-get install sylpheed (pour l'installer directement à partir d'internet)

Sans connexion internet, récupérez le paquet debian, par exemple sur http://packages.debian.org/unstable/mail/sylpheed.html, puis

dpkg -i sylpheedx.x.x-1i386.deb

ou

dpkg --install sylpheedx.x.x-1i386.deb

4.6 Installation sur FreeBSD

En utilisant l'outil /stand/sysinstall (c'est une interface basée sur les menus) :

Lancez /stand/sysinstall en tant que root.

Dans le menu principal, choisissez Configure -> Packages -> mail, puis sylpheed, puis retournez vers la liste des Packages, choisissez Install

Utiliser les portages

Allez chercher la version de Sylpheed portée sur FreeBSD à :

ftp://ftp.freebsd.org/pub/FreeBSD/ports/i386/packages-4.3-release/mail

puis, en tant que root : pkg-add sylpheed-0.x.x.tgz

En utilisant les portages avec compilation :

mettez d'abord votre portage à jour, traitez-le sur http://www.freebsd.org/ports/mail.html

cd /usr/ports/mail/sylpheed

make

Si cela réussit, executez make install

dans le cas où cela échoue, demandez le paquet requis, si nécessaire, (sur la même page), mettez le fichier du paquet dans /usr/ports/distfiles/ puis :

  • cd /usr/ports/mail/sylpheed
  • make
  • make install

Vous avez besoin d'une version plus à jour ?

Utilisez simplement la méthode standard d'Unix (veuillez voir « Éléments et points clés de l'installation »)
