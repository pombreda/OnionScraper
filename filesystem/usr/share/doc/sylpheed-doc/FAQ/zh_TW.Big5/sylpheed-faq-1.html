1.2 1.2 問：Sylpheed 是什麼？

答：Sylpheed 是一套可以用來收信及閱讀新聞群組的軟體，用 GTK+ 開發而成， 可在 X-window 系統上執行。這套系統設計的理念有：

  • 速度快
  • 介面友善而方便
  • 設定簡單，操作也十分直覺化
  • 功能十分豐富

1.3 1.3 問：我可以從哪裡取得 Sylpheed？

答：你可以從下列地方下載： sylpheed.good-day.net 在這裡可以下載程式碼的 tarball，以及包好的 RPM 套件（用於 RedHat 與 Mandrake 系統）、DEB 套件（用於 Debian 系統），另外還有用於 Solaris 與 FreeBSD 系統的套件。

1.4 1.4 問：Sylpheed 的作者是？

答：Hiroyuki Yamamoto(hiro-y@kcn.ne.jp)

1.5 1.5 問：Sylpheed 這個名字是什麼意思？

答：意思是「輕盈」（light weight），像風一樣。這是由「風之精靈」的名字 Sylphs 而來的。

1.6 1.6 問：Sylpheed 是怎麼儲存郵件的？

答：郵件是以 MH 格式儲存的。Maildir 與 mbox 格式尚未被支援， 不過現在已經可以將 mbox 格式匯入 Sylpheed 的郵件目錄了。

1.7 1.7 問：為何自動抓取郵件的功能不能運作？

答：自動抓取郵件的功能已經從 0.5.1 版起開始支援。

1.8 1.8 問：Sylpheed 有郵件過濾的功能嗎？

答：有，你可以從「設定」目錄中的「信件過濾設定」裡找到相關的設定。

不過 IMAP 的信件過濾功能尚未實作出來。

1.9 1.9 問：Sylpheed 可以處理 IMAP 郵件嗎？

答：可以。0.4.99 版之後即已支援。

1.10 1.10 問：Sylpheed 可以處理本機上的 unix 郵件信箱嗎？

答：可以。

1.11 1.11 問：Sylpheed 支援 IPv6 協定嗎？

答：可以。Sylpheed 完全支援 IPv6 的協定。

1.12 1.12 問：Sylpheed 支援 GPG 之類的加密協定嗎？

答：可以。Sylpheed 完全支援 GPG 協定，要啟動 GPG 功能請參考「安裝與設定」章節第 14 個問題。

1.13 1.13 問：Sylpheed 如何檢查 MIME 型態？

答：Sylpheed 利用 Mutt 的方式檢查 MIME 型態。

1.14 1.14 問：我可以用 Sylpheed 來寫 HTML 格式的信件嗎？

答：不行。有許多關於這方面的討論，最後大家的決議是這個功能並不需要。如果 你真的需要送 HTML 格式的信件，你也可以將一個網頁附加於信件中。另外有一 些修補（patch）可以幫助 Sylpheed 處理 HTML。你可以在 Sylpheed Patch Page 中 找到這些修補。（在這一頁的下面有相關連結）

1.15 1.15 問：Sylpheed 有沒有郵件地址與名稱補完（搜尋）的功能？

答：有。你想要從位址簿中找一個以 don 開頭的加入收件人， 你只需要在收件人一欄中輸入 don 然後按下 TAB 鍵。 如果位址簿中只有一個 don 開頭的人，它會完整地秀在收件人欄中。 若是不只一個，就會有一個選單讓你選擇你想要的人。

1.16 1.16 問：不過我按下 TAB 鍵時卻有錯誤訊息。

答：請確定你的位址簿中有這樣一個人。

1.17 1.17 問：Sylpheed 怎麼有辦法這麼快？

答：這就是它設計的理念之一，不是嗎？ http://www.thewildbeast.co.uk/sylpheed/ 並請參考安裝章節中，如何安裝修補程式的說明。

1.19 1.19 問：我要如何回報錯誤、送回自己作的修補程式，或者與其他人討論一些 Sylpheed 的事情？

答：要與別人討論 Sylpheed，你可以加入 Sylpheed mailing list： http://sylpheed.good-day.net。注意有英文與日文的 mailing list， 不要選錯了！若是你製作了一個修補程式而你打算把它送上來，請與 Sylpheed Patch homepage 的負責人：sylpheed@thewildbeast.co.uk 聯絡。

1.20 1.20 問：我送信時，信件標頭中表示我這封信的編碼是 US-ASCII>，但是我指定的是 ISO-8859-1？

答：如果信件中沒有任何一個超越 ASCII 的字元，（也就是 >= 0x80）， Sylpheed 會自動將標頭標記為 US-ASCII。

1.21 1.21 問：為何自動折行的功能無法運作？

答：折行功能是一個比較特殊的功能。你寫信時可以完全不分行， 但是你送出信件或送到暫存資料夾時，Sylpheed 會先將信件依據 你所設定的長度來折行，然後才送出信件。這其實比你想像中的還要 方便許多。折行之後如果你有在中間再加入一些東西，則下一行會被 移到新的一行去。 例如： 

     This is a long line that
     is wrapped.

現在你加入一個字在第一行，會變成： 

     This is a very long line
     that
     is wrapped

這可能會讓你得重新將你的信件排版。

不過在 Sylpheed 0.8.0 版以後，加入了較完整的自動排版折行功能。 你可以核取「一般設定」下的「撰寫郵件」中的「切割輸入過長的文字行」。

1.22 1.22 問：我要如何找到最新版的 Sylpheed FAQ？

答：請到 http://sylpheeddoc.sf.net 找 sylpheeddoc 專案。專案中提供了最新的 FAQ 與 Manual，並且都包含了多國語言版本。

1.23 1.23 問：我可以將 mbox 格式的信件匯入 Sylpheed 的 MH 信箱裡嗎？

答：可以。只要選擇你要匯入的信箱，然後從「檔案」目錄中選擇「匯入郵件」即可。

1.24 1.24 問：我可以用 procmail 來將我 MH 信箱中的郵件排序嗎？答：可以。只要確定你有在規則最後面加入 /.，讓 procmail 知道你在過濾 MH 信箱裡的郵件。例如：

0:

* ^Subject:.*sylpheed

sylpheedmail/.

你可以參考 man procmail。

1.25 1.25 問：當我更新 Sylpheed 時，我需要注意哪些事情？

答：注意以下幾件事：

更新到 0.4.50：你必須重新設定新聞群組。

更新到 0.4.63：你必須設定 URL 字串的顏色。

更新到 0.4.65：有些目錄已經被搬到別處了。

更新到 0.7.3：有許多熱鍵都更換了。

1.26 1.26 問：我要怎麼快速地更新 Sylpheed 的信件列表？

答：只要按下 Ctrl-Alt-U 或從「顯示」選單中選擇「更新信件列表」。

1.27 1.27 問：我為何無法使用 gvim 當做外部編輯器？

答：如果你要用 gvim 當外部編輯器，你必須使用 -f 參數。也就是說， 在設定時你的命令必須是：gvim -f %s 如果不這樣用，你在 離開編輯器之後，原先編的東西並不會加入到信件中來。

（感謝 Ed Collins 提供這個答案！）

1.28 1.28 問：我能不能在回覆時只引入部份的原信件內容？

答：可以。在觀看信件時，將您要回覆的部份標記起來然後選擇「回覆」。

1.29 1.29 問：我可以在回覆信件時選擇不引入原信件內容嗎？

答：可以。在「回覆」鍵上按右鍵，選擇「不引用原文」（「轉寄」鍵上也可使用）。

提示：您也可以用熱鍵設定來完成這個功能。

1.30 1.30 問：「選擇下載」要怎麼用？

答：有兩種作法：

1. 手動：選擇「工具/選擇下載」，並選擇「抓取」來取得信件標頭。接著，選擇要刪除的信件按下「刪除」。

2. 自動：產生一個整體的過濾規則，從伺服器上刪除不要的信件。接著如果您選擇「選擇下載」，所有符合這個規則的郵件都會被刪除。

注意：「選擇下載」預設只針對新的（未讀的）郵件。如果您要對已讀的郵件運作，您要選擇「取回所有郵件」。

1.31 1.31 問：為什麼會有兩個過濾的選項？

答：「信件過濾設定」是以前的，選項較少。另外新的過濾設定是 sylpheed-claws 中新增的，比較強力，將來可能會取代舊的功能。

提示：要將您舊的過濾規則應用到新的格式，您可以用 "filter_conv.pl" 這個小程式。這支程式可以在 sylpheed-claws 中的 tools 子目錄中找到。

1.32 1.32 問：我可以同時使用兩種過濾機制嗎？

答：不行，如果您使用了新的過濾機制，舊的就會被忽略。
